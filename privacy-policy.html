TrafficForge Privacy Policy

Developer: Ankur Pathak · Effective: May 2, 2026

TrafficForge is a Chrome extension for local traffic control and debugging. This policy describes how the extension handles information on your device.

Data handling

  • TrafficForge stores rule configuration locally in the browser using Chrome storage.
  • It may capture recent request metadata locally so you can review which requests matched your rules.
  • It does not intentionally upload traffic data, captured requests, or rule configuration to any remote server.
  • It does not include analytics, advertising, or third-party telemetry.

Permissions

TrafficForge requests browser permissions only to:

  • Apply user-defined request blocking and modification rules
  • Inspect request metadata needed for rule matching
  • Scope rules to the current tab when selected

Network traffic

TrafficForge can interact with browser-visible request traffic for supported protocols and request types. It operates according to the rules configured by you.

Third-party services

TrafficForge does not rely on third-party services for core functionality.

Contact

For questions about this privacy policy or how TrafficForge handles data, email ankur.logics@gmail.com. You can also use the support options shown on the TrafficForge listing in the Chrome Web Store.

Changes

This policy may be updated when the extension changes. The effective date at the top will be revised when material updates are published.
